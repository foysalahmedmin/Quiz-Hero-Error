What is the difference between Map(), forEach(), filter() and find()?

=> Map(), forEach(), filter() and find() are all array methods;

Map: The map method works on each element of the array separately and returns the result in a different array.
forEach: The forEach method also works on each element of the array separately, But does't create any array.
filter: The forEach method also works on each element of the array separately, but its function is to create a new array with certain elements from the array according to the condition.
find: The find method is similar to the filter method, but the find method return only one element from the array conditionally.

What is the difference between Local Storage, Session Storage and Cookies?

Local Storage: Local storage is useful for storing data that the user will need to access later, such as offline data. And this data will not be removed unless manually removed.
Session Storage: Session storage stores data in the browser much like local storage, but this data is not permanent. That data is gone on reload or close.
Cookies: Cookies also stor data in browser, and there are 2 types of cookies third party cookies and http cookies.

What is Event Loop?

The event loop is the secret behind JavaScript's asynchronous programming.
JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading. Let's take a look at what happens on the back-end.

The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.
